# Axe accessibility review — chatlab v1.1 patch

- **Date:** 2026-05-03
- **Branch:** fix/dependencies (post-security-sprint)
- **Tool:** `@axe-core/playwright` 4.x — WCAG 2.0 A/AA + WCAG 2.1 AA rules
- **Browser:** Chromium (Playwright headless, Desktop Chrome device)
- **Test file:** `test/e2e/axe.spec.ts`

This report closes **item 9** of the post-security-sprint action register (carried from item 7 of the rc-1 review). It supersedes the manual contrast-check artefact from [`2026-04-30-axe-contrast-check.md`](./2026-04-30-axe-contrast-check.md) for the ARIA / DOM-level checks.

## Findings before fix

Running against the pre-fix build revealed **2 critical violations**:

| Rule | Impact | Elements | Description |
| --- | --- | --- | --- |
| `aria-allowed-attr` | Critical | 4 `.tab` buttons in `App.tsx` + `LocaleToggle.tsx` | `aria-selected` is not a valid attribute for elements with implicit `role="button"`. |
| `aria-required-children` | Critical | `div.scroll-area[role="list"]` in `ChatList.tsx` | When the chat list is empty, the fallback hint `<div>` has no `role="listitem"`, violating the `role="list"` constraint. |

## Fixes applied

| File | Change |
| --- | --- |
| `src/ui/App.tsx` | Added `role="tablist"` to the top-level navigation `.tabs` div; added `role="tab"` to each navigation button. |
| `src/ui/components/admin/AdminPanel.tsx` | Added `role="tablist"` to the admin navigation `.tabs` div; added `role="tab"` to each button. |
| `src/ui/components/LocaleToggle.tsx` | Replaced `aria-selected` with `aria-pressed` — locale toggle buttons are toggle-buttons, not tabs; `aria-pressed` is the correct attribute for `role="button"`. |
| `src/ui/components/ChatList.tsx` | Added `role="listitem"` to the empty-hint `<div>` inside the `role="list"` container. |
